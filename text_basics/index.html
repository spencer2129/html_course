Text Basics

Much of the text content on a web page will be made of headings and paragraphs. Headings have a hiearchy. There can only be one <h1> in the page. After that they can have subtopics using <h2> elements. The subtopics can have subtopics using <h3> and so on.

The headers give semantic meaning. Screen readers and assistive technology can also navigate your page better if you're using headers to convey that this section is a subtopic of the main header, and so on.

The <hr> horizontal rule element doesn't add semantic meaning to the page, but visually it adds a horizontal line that can show that a new section or topic is about to start or just ended.

In HTML, white-space collapsing means that HTML does not honor all the white spacing you might add with a space bar or the enter key; it will not be represented in the page. You can use the line break <br> element inside a paragraph <p> element to add a new line inside the paragraph.

Block-level elements starts with a new line. They create space around the element. And they always take the full width available of its parent container.

Inline element does not start on a new line, and it only takes up as much width as necessary.
