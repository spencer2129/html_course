Links

Hyperlinks allow us to link documents to other documents or resources, link to specific parts of documents, or make apps available at a web address. Almost any content can be converted to a link, so that when it is clicked it will go to another web address.

A basic link is created by wrapping text or other content inside an <a> element and using the href attribute, also known as a Hypertext Reference, or target, that contains the web address.

Do not confuse the concept of linking to other documents or resourece in the internet with the element <link > which is use in the head element to pull resources from the internet or your webserver that are used in your site, such as an icon or a font from Google fonts.

<p>
  I'm creating a link to
  <a href="https://www.mozilla.org/en-US/">the Mozilla homepage</a>
</p>

A block-level element can be made into a link by wrapping the anchor (<a>) element around the block element.

<a href="https://developer.mozilla.org/en-US">
  <h1>MDN Web Docs</h1>
</a>

An image can be made into a link by wrapping the the anchor (<a>) around the image file referenced by the <img> element.

<a href="https://developer.mozilla.org/en-US">
  <img src="mdn_logo.svg" />
</a>

Add supporting information to your link by adding the title attribute title="this is the title" to the anchor element <a>.

<p>
  I'm creating a link to
  <a
    href="https:/www.mozilla.org/en-US"
    title="The best place to find more information about Mozilla's mission nad how to contribute">
    the Mozilla homepage</a>
</p>

A URL (Uniform Resource Locator) is a string of text that defines where something is located on the Web. URLs use paths to find files. Paths specify where the file you're interested in is located in the filesystem.

There are two types of URLs: absolute and relative

Absolute URL points to a location defined by its absolute location on the web, including protocol and domain name. For example, index.html page is uploaded to a directory called projects that sits inside the root of a web server, and the website's domain is https://www.example.com, then the page would be available at https://www.example.com/projects/index.html (or even just https://www.example.com/projects/ as most web servers just look for a landing page such as index.html to load if it isn't specified in the URL).

Relative URL points to a location that is relative to the file you are linking from. For example, if we wanted to link from our example file https://www.example.com/projects/index.html to a PDF file in the same directory, the URL would just be the filename -- project-brief.pdf-- (the equivalent absolute URL would be https://www.example.com/projects/index.html/project-brief.pdf).

<a href="./about.html">About me</a>

We can also create an internal link to a different part of the same web page; This is like creating a shortcut to diferent areas of the page. Make sure that add an element such as section with an id so you can link to it like this.

<nav>

    <li>
      a href="#projects">My Projects</a>
    </li>
  </ul>
</nav>

Link best practices

  • Use clear link working, such as <a href="https://www.mozilla.org/en-US/firefox" >Download Firefox</a>. Do not use wording like Click here.
  • Do not repeat the URL as part of the link text. URLs look ugly, and sound even uglier when a screen reader reads them out letter by letter.
  • Don't say "link" or "links" in the link text because it's just noise. Screen readers will tell people there is a link. Visual users will know there is a link because links are styled in a different color and underlined.
  • Keep your link text as short as possible -- this is helpful because screen readers need to interpret the entire link text.
  • Minimize instances where multiple copies of the same text are linked to different places. For example, multiple link texts such as "click here", "click here", "click here" in various parts of the web page.

Use the download attribute when linking to a download

<a href="./large-report.pdf" download>
  Download the sales report (PDF, 10MB)
</a>

<a
  href="https://download.mozilla.org/?product=firefox-latest-ssl&os=win64&lang=en-US"
  download="firefox-latest-64bit-installer.exe">
  Download Latest Firefox for Windows (64-bit) (English, US)
</a>

Open links in a new tab with the target attribute. Links by default open in the same tab as the page they're on, which allows the user to navigate back to the previous page using the browser's back button. However there are some times, such as when linking to external links, that you may choose to open links in a new tab.

<p>Firefox is developed by the
  <a href="https://www.mozilla.org/en-US/" target="_blank">Mozilla Foundation</a>
</p>

Email links -- when clicked, will open a new outgoing email message rather than linking to a resource or page.

<a href="mailto:me@example.com">Send me an email</a>

Make a call with links so that when clicked, it will open up the dialer.

<a href="tel:+123456789">My Phone</a>
