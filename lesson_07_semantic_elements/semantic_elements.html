HTML Semantic Tags

In programming, Semantics refers to the meaning of a piece of code. For example, what effect does running that line of JavaScript have? or what purpose or role does that HTML element have?

In JavaScript, consider a function that takes a string parameter and returns an <li> element with that string as its textContent. Would you need to look at the code to understand what the function did if it was called build('Peach'), or createLiWithContent('Peach')?

In CSS, consider styling a list with <li> elements representing different types of fruits. Would you know what part of the DOM is being selected with div > ul > li, or .fruits_item?

Semantics in HTML

Semantic elements give meaning to your web page. The word Semantic means that it provides meaning to something.

The three big areas in a web page are header, main, and footer

Html pages can have more than one header element. every social media post can have its own header.

There can only be one main element per web page.

You can have multiple footer elements in a page just like the header. For example, you might have a footer per social media post or article.

nav is also another semantic element. there can be more than one nav per page, for example like in the footer. when you have multiple navs you should label your navs. You can do that using aria-label="primary-navigation" as attribute in the nav element. If you had an h2 inside the nav, you can give an id="primary-navigation" to the h2 and in the nav use aria-labelledby="primary-navigation"

articles vs sections

use articles when you have a clear topic, where as section is more generic. For example inside a article you can have multiple sections.

aside element. mostly is seen as a sidebar to the page. But it can also be another section of a complementary text that is not as important as whats in the main or article. details and summary work together in a aside element

mark element is an inline element that is a highlighter.

time element to give semantic meaning to time and date.

you should avoid div and span elements when writing in HTML. Try to use only semantic elements. But ocasionally you might need to use divs when styling with CSS.
